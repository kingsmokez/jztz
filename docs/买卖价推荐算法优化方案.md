## 一、当前算法详解

### 1.1 代码位置

- 核心函数: `modules/scoring.py` 第1500-1678行 `calculate_buy_sell(stock, v5_score)`
- 技术指标: `modules/technical.py` `calculate_technical_indicators()`
- 回测脚本: `scripts/backtest_buy_sell.py`
- 数据模型: `modules/models.py` `StockQuote` / `FinancialData`

### 1.2 计算流程

```
输入: stock(股票数据), v5_score(五维评分)
  │
  ├─ Step 1: 估算合理PE
  │   avg_growth = (营收增速 + 利润增速) / 2
  │   growth_premium = min(avg_growth * 0.3, 15)
  │   fair_pe = ROE * 1.5 + growth_premium   → 限制在 [12, 60]
  │
  ├─ Step 2: 计算PE上涨空间
  │   if 当前PE < 合理PE:  pe_upside = (合理PE - 当前PE) / 当前PE
  │   else:                 pe_upside = 0
  │
  ├─ Step 3: 按评分分档确定买卖参数 (见下表)
  │
  ├─ Step 4: 技术面修正
  │   买入价: MA20*0.99 → MA60*0.98 → 硬底 price*0.85
  │   卖出价: recent_high*1.05 约束, boll_upper*1.05 约束
  │   止损价: ATR-based (2*ATR/price, 5-12%), 底部 buy*0.92
  │
  └─ Step 5: 输出 {buy, sell, stop_loss, position_pct, risk_reward_ratio, ...}
```

### 1.3 评分分档表

| v5_score | PE vs Fair PE | 买入折扣 | 上涨上限 | 推荐等级 | 星级 |
|----------|--------------|---------|---------|---------|------|
| >=82     | PE低估       | 3%      | 30%     | 强烈推荐 | 4-5  |
| >=68     | PE低估       | 5%      | 25%     | 推荐买入 | 3-4  |
| >=55     | PE低估       | 8%      | 20%     | 可逢低关注 | 2-3 |
| <55      | PE低估       | 10%     | 22%     | 轻度关注 | 1    |
| >=75     | PE高估*      | 10%     | 25%     | 等待更好买点 | 3  |
| >=58     | PE高估       | 12%     | 18%     | 高估观望 | 2    |
| <58      | PE高估       | 15%     | 15%     | 暂不推荐 | 1    |

*注: PE高估指 pe < fair_pe * 1.3

### 1.4 技术面修正规则

**买入价修正 (依次取最大值)**:
1. `buy_point = price * buy_discount` (基础折扣)
2. 若 `ma20 > buy_point` 且 `ma20 < price` → `buy_point = ma20 * 0.99`
3. 若 `ma60 > buy_point` 且 `ma60 < price` → `buy_point = ma60 * 0.98`
4. 硬底: `buy_point = max(buy_point, price * 0.85)`

**卖出价修正**:
1. `sell_point = price * (1 + upside)` (基础上涨空间)
2. 若 `recent_high > price` 且 `sell_point > recent_high * 1.05` → 取两者均值
3. 若 `boll_upper > price` 且 `sell_point > boll_upper * 1.05` → 取较小值
4. 最低要求: `sell_point > buy_point * 1.10`

**止损价**:
1. ATR模式: `stop_loss = price * (1 - 2*ATR/price)`, 限制5-12%
2. 无ATR时: `stop_loss = price * 0.92`
3. 地板: `stop_loss = max(stop_loss, buy_point * 0.92)`

**仓位建议**:
- 波动率 < 2%: 20%
- 波动率 2-4%: 15%
- 波动率 > 4%: 10%

### 2.4 核心问题诊断

**问题1: 卖出目标严重低估实际涨幅**
- 卖出触达率85%，远超合理区间(20-60%)
- 预测上涨空间17.7% vs 实际28.5%，偏差10.8%
- 后果: 投资者过早卖出，大量利润"留在桌上"

**问题2: 买入→止损路径占主导(57%)**
- 买入价触达后，57%的情况先触发止损再到达卖出目标
- 说明买入点设在技术面偏弱的位置(如MA20附近但趋势仍在下行)
- 后果: 看对方向但被止损洗出

**问题3: 合理PE公式过于简单**
- `ROE * 1.5 + growth_premium` 忽略了行业差异、利率环境、市场周期
- 银行ROE=15% → fair_pe=22.5，但银行实际PE常年在5-8倍
- 科技股ROE=12% → fair_pe=18，但科技股PE常在30-60倍

**问题4: 没有量价配合**
- 买卖价不考虑成交量确认
- 缩量跌破买入价(假破位)和放量跌破(真破位)应区别对待

**问题5: 折扣分档是静态的**
- 82分和81分的股票，买入折扣差2%，没有理论依据
- 牛市熊市用同样的折扣

**问题6: 卖出目标没有时间衰减**
- 不根据持有天数调整目标价
- 持有30天未达目标和持有3天未达目标，策略应不同

## 五、方案A: 规则微调(快速见效)

### 5.1 动态卖出目标(基于动量)

**当前代码** (`scoring.py` 第1530行附近):
```python
if v5_score >= 82:
    buy_discount = 0.97
    upside = min(max(pe_upside, 0.10), 0.30)  # 静态上限30%
```

**改为**:
```python
# 动量因子已由 mf_score_momentum() 计算，0-100分
momentum_score = stock.get("momentum_score", 50)

if v5_score >= 82:
    buy_discount = 0.97
    # 强动量 → 目标更高; 弱动量 → 目标更保守
    if momentum_score >= 70:
        upside = min(max(pe_upside, 0.20), 0.40)  # 强动量上限40%
    elif momentum_score >= 55:
        upside = min(max(pe_upside, 0.15), 0.30)  # 中等动量上限30%
    else:
        upside = min(max(pe_upside, 0.10), 0.22)  # 弱动量上限22%
```

**预期效果**: 卖出触达率从85%降至40-60%

### 5.2 量价配合买入点

**新增逻辑** (在计算 buy_point 之后):
```python
vol_ratio = stock.get("volume_ratio", 1.0)  # 当日量/20日均量

if vol_ratio < 0.5:
    # 极度缩量 → 破位可能为假信号，要求更深折扣
    buy_discount -= 0.02
elif vol_ratio > 2.0:
    # 放量突破 → 信号确认度高，可付溢价
    buy_discount += 0.01

buy_point = round(price * buy_discount, 2)
```

**预期效果**: 买入→止损率从57%降至<40%

## 六、方案B: 统计分位数买卖价(推荐)

### 6.1 核心思想

不猜"合理PE"，直接问历史数据: **这只股票过去120天，什么价位是低概率区(买入)、什么价位是高概率区(卖出)?**

**为什么比方案A更好**:
- 每只股票的买卖价由**自身历史分布**决定，不再一刀切
- 波动大的股票自动给出更宽的买卖区间，波动小的自动收紧
- 不依赖"合理PE"这个主观估算
- 评分只用来调整分位数位置，不再是硬分档

### 6.2 算法设计

```python
def calculate_buy_sell_statistical(stock, v5_score, tech_data=None):
    """基于历史价格分布的统计买卖价

    核心逻辑:
    - 买入价 = 历史价格分布的 P10-P25 分位数(取决于评分)
    - 卖出价 = 历史价格分布的 P75-P90 分位数(取决于评分)
    - 用评分调整分位数的具体位置
    """
    price = stock.get("price", 0)
    if price <= 0:
        return None

    # === 1. 获取历史K线，计算价格分布 ===
    closes = _get_historical_closes(stock["code"], days=120)
    if not closes or len(closes) < 30:
        # 数据不足，降级到旧模型
        return calculate_buy_sell(stock, v5_score)

    closes_arr = np.array(closes)

    # === 2. 计算关键分位数 ===
    p10 = np.percentile(closes_arr, 10)   # 近120天10%分位
    p25 = np.percentile(closes_arr, 25)   # 25%分位
    p50 = np.percentile(closes_arr, 50)   # 中位数
    p75 = np.percentile(closes_arr, 75)   # 75%分位
    p90 = np.percentile(closes_arr, 90)   # 90%分位

    # === 3. 用评分映射分位数 ===
    # 评分越高 → 买入分位数越高(愿意付更多) → 卖出分位数越高(目标更高)
    score_norm = min(max(v5_score, 40), 100) / 100  # 归一化到 0.4-1.0

    # 买入分位数: 高分股用P25(愿意付更多)，低分股用P10(要求更深折扣)
    buy_percentile = 10 + score_norm * 15  # 范围 16-25
    buy_point = np.percentile(closes_arr, buy_percentile)

    # 卖出分位数: 高分股用P90(目标更高)，低分股用P75
    sell_percentile = 75 + score_norm * 15  # 范围 81-90
    sell_point = np.percentile(closes_arr, sell_percentile)

    # === 4. 技术面微调 ===
    # RSI超卖时买入价可上浮，超买时下浮
    rsi = stock.get("rsi", 50)
    if rsi < 30:
        buy_point *= 1.02
    elif rsi > 70:
        buy_point *= 0.97

    # 量价配合: 放量时买入价可上浮
    vol_ratio = stock.get("volume_ratio", 1.0)
    if vol_ratio > 2.0:
        buy_point *= 1.01

    # MACD金叉确认: 买入价可上浮
    macd_signal = stock.get("macd_signal", "neutral")
    if macd_signal == "golden_cross":
        buy_point *= 1.01

    # === 5. 安全约束 ===
    buy_point = min(buy_point, price * 0.98)    # 买入价必须低于现价
    sell_point = max(sell_point, price * 1.05)   # 卖出价必须高于现价5%
    sell_point = max(sell_point, buy_point * 1.10)  # 至少10%利润空间

    # 止损: 基于ATR或买入价的5-8%
    atr = stock.get("atr", price * 0.03)
    stop_loss = price - 2 * atr
    stop_loss = max(stop_loss, buy_point * 0.92)

    # === 6. 仓位与风险收益比 ===
    volatility = _calc_volatility(stock)
    if volatility < 2:   position_pct = 20
    elif volatility < 4: position_pct = 15
    else:                position_pct = 10

    risk_reward_ratio = 0.0
    if price > stop_loss:
        risk_reward_ratio = round((sell_point - price) / (price - stop_loss), 2)

    # === 7. 五星评级 ===
    star_rating = 1
    if v5_score >= 86 and stock.get("roe", 0) >= 18:
        star_rating = 5
    elif v5_score >= 75:
        star_rating = 4
    elif v5_score >= 62:
        star_rating = 3
    elif v5_score >= 50:
        star_rating = 2

    return {
        "current": price,
        "buy": round(buy_point, 2),
        "sell": round(sell_point, 2),
        "stop_loss": round(stop_loss, 2),
        "position_pct": position_pct,
        "risk_reward_ratio": risk_reward_ratio,
        "upside": round((sell_point - price) / price * 100, 1),
        "downside": round((price - buy_point) / price * 100, 1),
        "recommendation": _get_recommendation(v5_score, buy_point, price),
        "star_rating": star_rating,
        "buy_reason": _get_buy_reason(rsi, vol_ratio, macd_signal),
        # 新增: 统计信息供前端展示
        "price_distribution": {
            "p10": round(p10, 2),
            "p25": round(p25, 2),
            "p50": round(p50, 2),
            "p75": round(p75, 2),
            "p90": round(p90, 2),
        },
    }
```

## 七、方案C: 蒙特卡洛模拟目标价

### 7.1 核心思想

基于股票的历史波动率和漂移率，模拟1000条未来价格路径，统计目标收益率的概率。给出概率化的建议: **"80%概率在30天内达到XX元"**。

### 7.2 算法设计

```python
def monte_carlo_target(stock, holding_days=30, n_simulations=1000):
    """蒙特卡洛模拟计算卖出目标价

    原理: 用历史日收益率模拟未来N天价格路径，
    找到"80%概率能达到"的卖出价

    Parameters:
        stock: 股票数据字典
        holding_days: 持有天数(默认30)
        n_simulations: 模拟次数(默认1000)

    Returns:
        (buy_target, sell_target, confidence_info)
    """
    closes = _get_historical_closes(stock["code"], days=60)
    if not closes or len(closes) < 20:
        return None

    # 计算日收益率统计量
    returns = np.diff(np.log(closes))
    mu = np.mean(returns)       # 日均收益率(漂移率)
    sigma = np.std(returns)     # 日波动率

    price = stock.get("price", 0)
    if price <= 0:
        return None

    # 模拟1000条路径
    final_prices = []
    path_max_prices = []  # 每条路径的最高价
    path_min_prices = []  # 每条路径的最低价

    for _ in range(n_simulations):
        random_returns = np.random.normal(mu, sigma, holding_days)
        cum_returns = np.cumsum(random_returns)
        prices = price * np.exp(cum_returns)

        final_prices.append(prices[-1])
        path_max_prices.append(np.max(prices))
        path_min_prices.append(np.min(prices))

    final_prices = np.sort(final_prices)
    path_max_prices = np.sort(path_max_prices)
    path_min_prices = np.sort(path_min_prices)

    # P80卖出价: 80%的模拟路径最终价能达到
    sell_80 = np.percentile(final_prices, 80)
    # P80路径最高价: 80%的模拟路径中曾达到的最高价
    sell_peak_80 = np.percentile(path_max_prices, 80)
    # P20路径最低价: 只有20%的模拟路径会跌到
    buy_20 = np.percentile(path_min_prices, 20)

    # 置信度信息
    confidence = {
        "sell_80_prob": f"80%概率{holding_days}天内达到{sell_80:.2f}元",
        "sell_peak_80_prob": f"80%概率{holding_days}天内触及{sell_peak_80:.2f}元",
        "buy_20_prob": f"仅20%概率{holding_days}天内跌至{buy_20:.2f}元",
        "simulated_return_mean": f"{(np.mean(final_prices)/price - 1)*100:.1f}%",
        "simulated_return_std": f"{np.std(final_prices)/price*100:.1f}%",
    }

    return buy_20, sell_peak_80, confidence
```

### 7.3 与方案B的结合

```python
def calculate_buy_sell_v4(stock, v5_score, tech_data=None):
    """买卖价 V4: 统计分位数 + 蒙特卡洛验证"""
    # 先用统计分位数得到基础买卖价
    result = calculate_buy_sell_statistical(stock, v5_score, tech_data)
    if not result:
        return calculate_buy_sell(stock, v5_score)

    # 用蒙特卡洛验证卖出价是否合理
    mc_result = monte_carlo_target(stock, holding_days=30)
    if mc_result:
        buy_mc, sell_mc, confidence = mc_result

        # 如果统计分位数的卖出价远低于蒙特卡洛P80，
        # 说明还有上涨空间，可适当提高
        if result["sell"] < sell_mc * 0.85:
            result["sell"] = round((result["sell"] + sell_mc) / 2, 2)
            result["upside"] = round((result["sell"] - result["current"]) / result["current"] * 100, 1)

        # 添加概率信息
        result["monte_carlo"] = confidence

    return result
```

## 八、方案D: Kelly公式动态仓位

### 8.1 核心思想

当前仓位建议(10%-20%)只看波动率，没有考虑胜率和赔率。Kelly公式根据胜率和赔率计算最优仓位比例。

### 8.2 算法设计

```python
def kelly_position(win_rate: float, avg_win: float, avg_loss: float) -> float:
    """Kelly公式计算最优仓位比例

    f* = (p * b - q) / b
    p = 胜率, q = 1-p, b = avg_win / avg_loss (赔率)

    实际使用半Kelly(更保守):
    position = f* / 2

    Parameters:
        win_rate: 历史胜率 (0-1)
        avg_win: 平均盈利比例 (如 0.18 表示18%)
        avg_loss: 平均亏损比例 (如 0.07 表示7%)

    Returns:
        建议仓位比例 (0-0.25)
    """
    if avg_loss <= 0 or win_rate <= 0:
        return 0.10  # 默认10%

    b = avg_win / avg_loss  # 赔率
    q = 1 - win_rate
    f = (win_rate * b - q) / b
    f = max(0, min(f, 0.25))  # 上限25%

    return round(f / 2, 2)  # 半Kelly，更保守
```

    """Kelly仓位替代当前波动率仓位"""
    # 确定评分档位

```python
def auto_tune_params():
    """每周运行: 根据回测结果自动微调参数

    调参规则:
    - 卖出触达率 > 65% → 提高上涨上限
    - 卖出触达率 < 25% → 降低上涨上限
    - 买入→止损率 > 45% → 收紧买入折扣
    - 买入→止损率 < 20% → 放宽买入折扣
    - 预测vs实际偏差 > 10% → 调整分位数基准
    """
    result = backtest_current_picks(top_n=50)
    summary = result["summary"]

    params = load_adaptive_params()

    # 卖出触达率调整
    if summary["sell_rate"] > 0.65:
        params["upside_cap_offset"] += 0.03
        params["sell_percentile_base"] += 2
    elif summary["sell_rate"] < 0.25:
        params["upside_cap_offset"] -= 0.03
        params["sell_percentile_base"] -= 2

    # 买入→止损率调整
    buy_stop_rate = summary.get("buy_stop_rate", 0.5)
    if buy_stop_rate > 0.45:
        params["buy_discount_offset"] -= 0.01
        params["buy_percentile_base"] -= 1
    elif buy_stop_rate < 0.20:
        params["buy_discount_offset"] += 0.01
        params["buy_percentile_base"] += 1

    # 预测偏差调整
    upside_gap = abs(summary["avg_upside_pred"] - summary["avg_upside_actual"])
    if upside_gap > 10:
        params["sell_percentile_base"] += 3

    # 参数边界约束
    params["buy_percentile_base"] = max(5, min(params["buy_percentile_base"], 30))
    params["sell_percentile_base"] = max(65, min(params["sell_percentile_base"], 95))
    params["upside_cap_offset"] = max(-0.10, min(params["upside_cap_offset"], 0.15))
    params["buy_discount_offset"] = max(-0.05, min(params["buy_discount_offset"], 0.05))

    # 记录历史
    params["last_updated"] = datetime.now().strftime("%Y-%m-%d")
    params["update_history"].append({
        "date": params["last_updated"],
        "action": "auto_tune",
        "sell_rate": summary["sell_rate"],
        "buy_stop_rate": buy_stop_rate,
        "upside_gap": upside_gap,
    })

    # 只保留最近20条历史
    params["update_history"] = params["update_history"][-20:]

    save_adaptive_params(params)
    return params
```

### 12.2 降级策略

```python
def calculate_buy_sell_v3(stock, v5_score, tech_data=None):
    """买卖价 V3: 统计分位数 + 降级保护"""
    try:
        result = calculate_buy_sell_statistical(stock, v5_score, tech_data)
        if result and result["buy"] > 0 and result["sell"] > result["buy"]:
            # 合理性检查
            discount = (stock["price"] - result["buy"]) / stock["price"]
            if 0.01 <= discount <= 0.30:  # 折扣在1%-30%之间
                return result
    except Exception:
        pass

    # 降级到V2(当前模型)
    return calculate_buy_sell(stock, v5_score)
```
